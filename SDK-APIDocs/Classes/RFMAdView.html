RFMAdView Class Reference

Inherits from UIView
Conforms to UIGestureRecognizerDelegate
Declared in RFMAdView.h

Overview

RFMAdView class provides the view for displaying Revv For Mobile ads.

Use one of the create functions to create a banner or interstitial ad. We recommend to create a view once during the view controller creation and then use only the requestFreshAd method to refresh the view with new ads.

applyVerticalOffsetForiOSGreaterThan:portraitOffset:landscapeOffset:

Adjust the placement of ad view coordinates contionally on specific OS versions

- (void)applyVerticalOffsetForiOSGreaterThan:(NSString *)iosVersion portraitOffset:(CGFloat)portraitOffset landscapeOffset:(CGFloat)landscapeOffset

Parameters

iosVersion

Lowest OS version that the offset should be applied to. Set iOS version to string value of numerical version . ex: @“7.0”.

portraitOffset

Vertical offset to be applied on RFMAdView coordinates in portrait mode

landscapeOffset

Vertical offset to be applied on RFMAdView coordinates in landscape mode

Discussion

Optional method to adjust the adview placement if parent view controller cannot set the edges for extended layout to none. This API will be useful for allowing apps running OS lower than iOS7 to set their placement coordinates for iOS6 and iOS7+ without an explicit version check. Call this API for with the correct offset values to ensure that the adview is created at the desired coordinates.

requestCachedAdWithRequestParams:

Request a new cacheable ad from RFM ad server

- (BOOL)requestCachedAdWithRequestParams:(RFMAdRequest *)requestParams

Parameters

requestParams

Request Parameters for this call. Instance of RFMAdRequest

Return Value

Status Boolean status code informing if the request was accepted by the SDK or not.

Discussion

Method to request a cacheable ad once the AdView has been created. You can call this several times during the view lifecycle. The returned status shows whether the request was accepted by the SDK. For example, if this method is called when the user is viewing the landing page of a previous ad, the SDK does not initiate a new ad request and instead returns BOOL “NO” as status.

Warning: A cacheable ad has a two step load process. Not all ads supported on RFM are cacheable. Please contact your Account Manager if you plan on using this method.
